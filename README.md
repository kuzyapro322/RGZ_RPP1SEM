# РГЗ: WEB-приложение для учета финансов

Вариант индивидуального задания: **7 — удаление операции**.

## Запуск

Перед первым запуском нужно создать локальный `.env` на основе примера:

```powershell
Copy-Item .env.example .env
```

В `.env` хранятся пароль PostgreSQL, секретный ключ Flask-сессий и статические курсы валют. Этот файл не коммитится в Git.

```powershell
docker compose up --build
```

После запуска:

- WEB-приложение: http://localhost:5000
- Внешний сервис курса валют: http://localhost:5001/rate?currency=USD

## Что реализовано

- регистрация пользователя `/reg`;
- авторизация и выход;
- добавление финансовой операции `/add_operation`;
- просмотр операций `/operations?currency=RUB|USD|EUR`;
- конвертация операций через внешний сервис курса валют;
- индивидуальный вариант 7: удаление операции через `POST /delete_operation/<operation_id>`;
- ограничения в PostgreSQL: тип операции только `income`/`expense`, сумма больше 0;
- каскадное удаление операций пользователя на уровне внешнего ключа БД;
- нормализация логина и базовая проверка сложности пароля;
- PostgreSQL в Docker;
- Jinja-шаблоны для страниц приложения.
